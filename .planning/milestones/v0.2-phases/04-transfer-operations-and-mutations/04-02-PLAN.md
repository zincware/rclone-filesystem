    Add mutation invalidation tests after the existing cache tests. All rclone_fs only (cache invalidation is an implementation detail, not an s3fs comparison):

    1. `test_put_invalidates_cache(s3_base, rclone_fs, tmp_path)`: Create bucket + file via s3_base. Call `rclone_fs.ls(bucket)` to populate cache. Assert `bucket in rclone_fs.dircache` (populated). Put a new file via `rclone_fs.put(local_file, f"{bucket}/newfile.txt")`. Assert `bucket not in rclone_fs.dircache` (invalidated). Call `rclone_fs.ls(bucket)` again and verify new file appears.

    2. `test_rm_file_invalidates_cache(s3_base, rclone_fs)`: Create bucket + file. `rclone_fs.ls(bucket)` to populate cache. Assert `bucket in rclone_fs.dircache`. `rclone_fs.rm_file(f"{bucket}/file.txt")`. Assert `bucket not in rclone_fs.dircache`. `rclone_fs.ls(bucket)` no longer shows the file.

    **Key assertion pattern:** Use `assert "{path}" not in rclone_fs.dircache` to verify invalidation, and `assert "{path}" in rclone_fs.dircache` to verify cache was populated before the mutation.
