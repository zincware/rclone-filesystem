   - Remove `import os` (no longer needed for env vars)
   - In `s3_base` fixture, use `pytest.MonkeyPatch.context()` as context manager (works with module scope):
     ```python
     with pytest.MonkeyPatch.context() as mp:
         mp.setenv("AWS_SECRET_ACCESS_KEY", "foo")
         mp.setenv("AWS_ACCESS_KEY_ID", "foo")
         mp.delenv("AWS_PROFILE", raising=False)
         yield ...
     ```
   - This ensures env vars are restored after the module-scoped fixture tears down

3. **rclone-python API for remote setup (user decision from CONTEXT.md):**
   - Add imports: `from rclone_python import rclone` and `from rclone_python.utils import run_rclone_cmd`
   - Remove `import subprocess`
   - `setup_rclone_remote` fixture needs `endpoint_uri` -- make `s3_base` yield both the boto3 client AND the endpoint_uri (as a tuple or use a separate fixture)
   - Better approach: make `s3_base` yield the client, and store `endpoint_uri` in a module-level variable set inside `s3_base`, OR create a separate `endpoint_uri` fixture
   - Recommended: have `s3_base` yield a dict or namedtuple with `client` and `endpoint_uri`, OR yield just endpoint_uri and create client separately
   - Simplest: `s3_base` yields `endpoint_uri` (the client is only used for bucket setup in tests that need it, and they can create it from endpoint_uri)
   - Actually, looking at test usage: tests use `s3_base` as a fixture parameter name. Keep backward compat by yielding the boto3 client, and pass endpoint_uri via a closure or module var.
   - Best approach: create a module-level variable `_endpoint_uri = None`, set it in `s3_base`, reference it in `setup_rclone_remote`. This is clean and simple.
   - In `setup_rclone_remote`:
     ```python
     if rclone.check_remote_existing("s3-test"):
         run_rclone_cmd('config delete "s3-test"')
     rclone.create_remote("s3-test", "s3",
         env_auth="false", access_key_id="foo",
         secret_access_key="foo", endpoint=_endpoint_uri,
         force_path_style="true", acl="private")
     yield
     run_rclone_cmd('config delete "s3-test"')
     ```
