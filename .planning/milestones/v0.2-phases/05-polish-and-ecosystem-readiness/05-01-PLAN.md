    - Constructor: show_progress=None (default) falls back to settings.show_progress
    - put_file: resolves show_progress via kwargs.pop("show_progress", self._show_progress), passes to rclone.copyto
    - put_file: resolves pbar via kwargs.pop("pbar", None), passes to rclone.copyto

    - cp_file: same pattern, but also pops pbar from kwargs and passes through
    - RCloneFile.__init__: stores self._show_progress = fs._show_progress; passes to rclone.copy for read mode download
    - RCloneFile.close: passes self._show_progress to rclone.copyto for write mode upload
  </behavior>
  <action>
1. In settings.py: Add `show_progress: bool = False` field to RCloneFileSystemSettings.

2. In __init__.py RCloneFileSystem.__init__: Add `show_progress=None` parameter. Resolve using same pattern as temp_dir: `self._show_progress = show_progress if show_progress is not None else settings.show_progress`.

3. In put_file: Replace `show_progress=False` with resolved value. Use `kwargs.pop("show_progress", self._show_progress)` and `kwargs.pop("pbar", None)`. Pass both to rclone.copyto explicitly: `rclone.copyto(lpath, rclone_path, show_progress=show_progress, pbar=pbar, **kwargs)`.

4. In get_file: Same pattern as put_file. Pop show_progress and pbar from kwargs, pass to rclone.copyto.

5. In cp_file: Same pattern. Pop show_progress and pbar from kwargs, pass to rclone.copyto. Note: cp_file currently does not accept **kwargs -- add **kwargs parameter and pop show_progress/pbar from it.

7. In RCloneFile.close: Change `rclone.copyto(..., show_progress=False)` to `rclone.copyto(..., show_progress=self._show_progress)`.

IMPORTANT: Default must remain False everywhere. Only explicit opt-in enables progress. Pop show_progress/pbar from kwargs BEFORE forwarding to rclone to avoid "got multiple values" TypeError.
